Metodo POST y GET para envio de datos.

GET

  • Se utliza para solicitar datos del servidor.
  • Los datos del formulario se agregan a URL como parametros de consulta
  • Es visible en la barra de direcciones del navegador y puede ser marcado o guardado en el navegador
  • Es adecuado para solicitudes que no cambian el estado del servidor (consulta de base de datos y busqueda)
  • Los datos tienen una longitud limitada por las restricciones de longitud URL

POST

  • Envia datos al servidor para procesarlos.
  • Los datos del formulario no son visibles.
  • Es adecuado para enviar datos confidenciales y ademas pueden ser mas extensos.
  • No se utliza para solicitudes que cambian el estado del servidor de ser asi se debe usar con precaucion.
  • No tiene restriccion de longitud como el metodo GET
