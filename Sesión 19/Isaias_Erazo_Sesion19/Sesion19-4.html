Metodo POST y GET para envio de datos

GET

  • Se utiliza para solicitar datos del servidor.
  • Los datos del formulario se agregan a la URL como parametros de consulta.
  • Es visible en la barra de direcciones del navegador y puede ser marcado en el navegador.
  • Es el adeacuado para solicitudes que no cambian el estado del servidor (Consulta de bases de datos y busqueda).
  • Los datos tienen una longitud limitada por las restricciones de longiitud URL.

POST

  • Envia datos al servidor para procesarlos.
  • Los datos del formulario son visibles.
  • Es adecuado para enviar datos confidenciales y ademas pueden ser mas extensos.
  • No se utiliza para solicitudes que cambian el estado del servidor de ser asi se debe usar con precaucion.
  • No tiene restricion de longitud como el metodo GET.
